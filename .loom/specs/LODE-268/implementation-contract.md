# LODE-268 Implementation Contract

The public runtime input is only an opaque `detail_ref`. Core owns persisted provenance, freshness, single-use enforcement, internal material resolution, and BOSS `securityId` / `encryptJobId`; the BOSS `0.1.1` normalized public output requires only the opaque `detail_ref` binding and rejects raw security identifiers. Harbor owns browser, session, page, and evidence facts. Lode pins both package output schemas and requires complete normalized public fields, a non-empty bounded summary, and source/evidence binding for every public field. Raw DOM, network bodies, xsec tokens, cookies, tokens, and profile material remain forbidden. Lode runs neither component.
